## 一、世界观基调：表面现代都市 × 地下混沌

### 1.1 核心设定：信息混沌

**超凡事件过去3周后，关于它的一切仍然没有在公共舆论中形成明确的叙事。**

不是政府有能力"全面封锁"——而是政府自己也处在混乱中：
- 不同地区对地下城降临的应对完全不同。A市可能封路拉警戒线，B市可能直接无视，C市可能派了警察围了一圈不知道该怎么办
- 删帖是平台的自发行为，不是统一指令——平台删帖是因为"这种内容太离谱了，怕惹事"，但尺度差异极大
- 视频照片已经满天飞了——朋友圈、群聊、外网，根本删不完
- 不存在"觉醒者被管控"——政府连觉醒者是什么都不清楚，更谈不上管理
- 唯一形成默契的是"官方不公开承认"。各级政府的口径出奇一致——不回应、不承认、不否认

结果是：**普通人处于"半信半疑"的状态。**

有人刷到了视频 → 觉得"应该是真的吧"
官方不回应 → "也可能是假的，不然怎么没人管"
朋友发了截图 → "到底是不是P的？"
第二天刷微博 → 热搜没了，帖子没了 → "算了，可能确实没什么"

**这是一种信息混沌，不是信息封锁。** 没人能确认发生了什么，但也没人能否认发生了什么。

这个设定带来了三个关键叙事效益：
1. **主角的双重生活真实可信**——白天他是普通大学生，没人知道他觉醒了
2. **地下城竞争压力可控**——不是全民涌入，只有极少数觉醒者在秘密行动
3. **独家情报的价值被放大**——你知道某个地方有地下城而别人不知道，这就是先发优势

### 1.2 混乱应急的表现

这种信息混沌的产生，不是因为有高效的管控系统，而是因为没有系统能够完整地处理这一切：
- 政府内部也在震惊和恐慌——"地下城是什么？？怎么会有怪物？"
- 各地方政府的应对完全不同——有的果断封锁、有的无能为力、有的装作没看到
- 网络上的信息潮水般涌来又被平台掐断，但总有人在转发和存档
- 觉醒者之间没有统一的社交连接——零散地通过极少数幸存论坛/群组联系
- 最重要的：**地下城降临到不同地点的间距极不规律，有的降临在城市中心、有的在无人郊区，信息暴露的差异巨大**
- 觉醒者圈子是小范围地下社群——"你知道谁是觉醒者吗？我不知道，我自己就是觉醒者"

**写作时的分寸：**
- 新闻上不会出现"地下城降临"的报道
- 但同学之间可能会有"你听说了吗？某栋楼下面好像塌了出事了"的传闻
- 觉醒者之间靠极少数幸存论坛/私密群组联系——"我号差点被封了，这地方居然还有活人"
- 不要写成"政府全面封锁"——要写成"所有人都不知道该怎么办"

### 1.3 这不是一个"成熟的世界"

**开局的核心氛围基调是：**

> 全世界都还没搞清楚发生了什么。觉醒者是零散的。地下城是突然出现的。没有体系，没有规则，没有情报共享，没有定价标准，没有官方管理机构。

**写作时反复检查的点：**

| 违禁项 | 错在哪 | 正确姿势 |
|--------|--------|---------|
| "觉醒者管理局"以官方身份出现 | 初期不存在任何官方管理觉醒者的机构 | 警察+地方官员临时搭台子，一脸懵逼地围住地下城入口 |
| "地下城交易行"或"拍卖会"出现 | 初期没有定价体系，没有成规模的市场 | 私下交易，以物易物，双方靠直觉估价 |
| 角色说"这个职业很稀有" | 职业体系不成熟，没人能判断稀有程度 | 角色只能凭感觉说"我没见过这个职业"，而不是铁口直断 |
| 角色说"这装备值X块" | 没有市场价 | 角色只能说"我觉得这应该值点钱吧"或"你想换什么" |
| 情报贩子出现 | 没有情报市场 | 口耳相传，论坛帖子，朋友圈转发 |
| 觉醒者摆摊交易 | 封锁环境下不存在成规模的交易活动 | 自用、熟人私下交换、留着 |

### 1.4 半数据化的表现方式

世界是半数据化的——觉醒者能看到面板、经验值、技能树，但这些数据和真实超凡能量的关系没人弄得清楚。

**写作指导：**
- 角色讨论数据时带有猜测性质——"我怀疑这个技能的伤害是这么算的"而不是"技能说明写了"
- 面板提供的信息被角色视为参考而非真理——"系统显示我升到2级了，但感觉也没什么变化啊"
- 不同职业者的面板格式可能不同——"你的面板怎么没有被动技能栏？我的有啊？"
- 普通人完全看不到这些数据——他们只能看到觉醒者"念了一下咒就有火球飞出去"

### 1.5 信息差的运用

初期最大的叙事资产是**所有人都处于信息弱势**。主角比其他人知道得多（前世记忆），但也没多到能全知全能。

**信息层级表（前20章）：**

| 信息类型 | 主角知道吗 | 其他觉醒者知道吗 | 普通人知道吗 |
|---------|:--------:|:--------------:|:-----------:|
| 地下城是什么 | 懂概念+懂框架（前世经验） | 知道"有个诡异空间"，但不知道名称和机制 | 大部分不知道，少数见过的人也不清楚 |
| 怎么打地下城 | 懂框架+有塔防经验 | 不知道，在摸索中 | 不知道 |
| 觉醒后怎么升级 | 懂框架（前世记忆） | 知道"杀怪能变强"，但不清楚数据细节 | 不知道 |
| 源质是什么 | 懂概念但不知全貌 | 完全不知道 | 完全不知道 |
| 装备价值 | 知道"游戏里这类值钱"，但不清楚蓝星行情 | 知道"装备应该值点钱但不知道值多少" | 不知道 |
| 职业体系 | 懂类型分布 | 只知道"我被分配了XX职业"，可能见过1-2个职业 | 不知道 |
| 地下城数量 | 不知道具体数字 | 只知道本地的（自己觉醒的那个） | 极少数人听说过"有很多"但没概念 |

**叙事运用**：信息差让主角能做对的选择而不显得开挂——他只是比其他人多了一点概念框架，具体操作和应对全要靠他自己。

### 1.6 觉醒者社交链的特殊路径

在信息混沌的环境下，觉醒者之间不存在公开的社交链。他们的连接方式极为受限：

**路径①：线下偶遇（偶然性极高）**
- 两个觉醒者恰好在同一个地下城出现
- 或现实活动中观察对方的异常表现（"你上次躲闪那次，普通人不可能做到"）
- **这种偶遇既是故事的亮点，也是制造张力的锚点**

**路径②：暗语触发**
- 少数觉醒者在公共论坛上用暗语发帖试探（"有人去过那个地方吗？就是那个我打了一晚上怪的地方"）
- 这种帖子往往在发出后几分钟内被删除，只有恰好在线且知道含义的人能捕获
- **每次暗语接触都是冒险——你不知道回复你的人是真的觉醒者还是平台的人**

**路径③：口碑传播**
- 觉醒者A认识了觉醒者B，B介绍了C
- 形成极小圈子的熟人网络
- **这种网络在初期极为零散，觉醒者之间可能相隔几个城市的空间距离**

**主角的起步情况**：
- 刚觉醒时，主角没有任何觉醒者社交圈（和其他人一样）
- 他的前世记忆只能告诉他"存在其他觉醒者"和"你要小心暴露"，但无法告诉他"蓝星上的觉醒者在哪"
- 初期的人际连接只能通过上述三条路径逐步建立

### 1.7 初期的装备/资源流通模式

在封锁/信息混沌/觉醒者零散分布的环境下，装备和资源的流转只有三种可能渠道：

**渠道①：自用消化（绝大多数情况）**
- 打到装备自己用，打到材料先攒着
- 这是初期最主流的处理方式

**渠道②：熟人交换（极小圈子）**
- A和B互相认识且信任
- "我这把剑你用不错，你那件护甲我刚好需要——换？"
- 没有货币、没有定价、只有"你觉得值不值"

**渠道③：偶然性"寄售"（极度偶然，极少发生）**
- 觉醒者A恰好看了一个帖子，发现发帖人B好像也是觉醒者
- 私聊试探，确认后提出"我有个东西，你感兴趣吗？"
- 这种交易的风险极高——你怎么确定对方不是政府的人？不是想打劫你的人？

**关于"地下城门口自发摆摊"：**
- ❌ 不应出现。地下城入口被围/警戒/封锁状态中，摆摊会被城管清场
- ✅ 可以出现的场景：地下城入口附近的网吧/小吃店里，觉醒者在角落碰头
- 这不是"交易市场"——这时两三个人私下交换物品而已

## 二、主角双线节奏指南

### 2.1 降临期 ↔ 冷却期

这是主线节奏的**核心切换规则**：

| 阶段 | 状态 | 主角活动 |
|------|------|---------|
| **降临期** | 地下城在蓝星实体存在 | 以核心分身身份经营城+以法师身份进出自如 |
| **冷却期** | 地下城沉睡在混沌中 | 准备下次降临、处理突发意外、以法师身份活动 |

**降临期特征：**
- 有限窗口，倒计时感强
- 主角要同时兼顾两件事：城内操控 + 现实活动
- 外人可以进他的城 → 制造冲突和博弈
- 降临结束前有紧张感——入口接近关闭，有没有人还困在里面？

**冷却期特征：**
- 安全窗口——别人进不了他的城
- 准备为主：恢复源质、调整布局、升级核心分身
- 但也可能有**突发意外**（详见§2.2）
- 冷却是"暴风雨前的宁静"，不是"无事发生"

### 2.2 冷却期活动优先级

| 优先级 | 活动 | 说明 |
|-------|------|------|
| **主线①** | 准备下次降临 | 恢复源质、调整布局、补充怪物、优化难度曲线 |
| **主线③** | 突发意外 | 冷却期不是安全的——可能有人追踪降临点、有势力开始查主角、地下城内出现预料外的问题 |
| **辅线②** | 偶尔埋伏笔 | 在现实世界活动中顺带埋下线索——碰到某个关键人物、听到某个信息、收到意外的消息 |
| **空闲** | 法师面板升级 | 冷却期主角以法师身份正常活动，打怪获取经验 |

**节奏示意：**

```
降临期 → [冷却前段：准备] → [冷却中段：突发] → [冷却后段：收线+埋钩子] → 下一次降临
```

**每个冷却期至少包含：**
- 1次明确的"准备行动"（主线①）
- 0~1次"突发意外"（主线③）——不是每次冷却期都要有，但出现时不能突兀
- 1次"埋钩子"（辅线②）——哪怕只是一句话

**突发意外的合理触发条件：**

每个"突发意外"必须有明确的前置理由。

| 意外类型 | 触发条件 | 示例 |
|---------|---------|------|
| 被人追踪降临点 | 主角的地下城降临在某人的房产底下、或降临位置在人多的地方被目击了 | "刚才那个塌陷的地方好像有人进去了" |
| 有势力开始查主角 | 主角在现实活动中过于反常，被有心人关注 | "我发现你最近总在同一个地方绕路走" |
| 地下城内部异常 | 核心分身升级过程中出现了预料外的连锁反应 | "第三层的墙壁自己裂开了" |
| 其他觉醒者闯入 | 巧合——对方在探索其他地下城时，跑到了主角的城门口 | 概率极低（初期不建议用，容易显得刻意） |

**在冷却期中塞"突发意外"之前，先问自己三个问题：**
1. 这个意外有没有前置条件？（某人看到了？有人追踪到了？）
2. 这个前置条件在前文中有没有铺垫？（上一章提过一句"有人盯着他"？）
3. 如果没有铺垫，那它是一次"天降横祸"还是一次"信息差造成的冲突"？（后者可以）

### 2.3 双线并行原则

| 原则 | 说明 |
|------|------|
| **不同时高强度** | 法师线和核心线不会在同一章中同时进行高强度操作。如果一章是战斗章（后台操控迎战入侵者），本章的法师活动就是轻量的（上课、吃饭、日常对话）；如果一章以法师身份活动为主，核心线就退到后台监控。 |
| **切换有成本** | 两边的切换在叙事上需要"过渡"——不是上一段还在打怪下一段就开始调陷阱。过渡可以是走路上、睡觉前、发呆时。 |
| **互相服务** | 两条线不是割裂的。法师的现实活动（上课、社交、网络搜索）获取的情报和资源服务于核心经营；核心经营的产出（源质、装备）是法师前20章唯一的升级路径。 |

### 2.4 时间分配基线

**前20章的时间密度必须让读者觉得合理。** 主角身兼：大学生日常 + 核心分身经营地下城 + 冷却期准备 + 突发意外应对 + 论坛互动。

**大学生活占用的基线：**
- 课程占用：每天6-8小时（含上课+自习+作业）
- 可压榨时间：课余+晚上+周末
- 必要的"露脸"场景：上课点到、宿舍出现、食堂吃饭、偶尔和室友互动
- 不必要的细节可省略（如10点上课还是11点上课这种没意义的描写直接过）

**地下城经营的占用：**
- 降临期：每天2-4小时高强度投入（事后写作要体现"他今天打了一架又被同学说怎么这么累"）
- 冷却期：每天30分钟-1小时（只有后台调整，不影响现实生活）

**法师活动的占用（前20章）：**
- 前20章不存在"法师去别人地下城打怪"——主角找不到其他地下城
- 法师身份在前20章主要负责：上课签到、校园社交、网络情报收集、处理现实层面的突发意外
- 法师战斗能力的升级，在前20章主要依赖**核心经营产出的源质转化**，而非主动打怪

**时间分配的底线法则：**
1. 主角会熬夜，但次数有限（不能天天熬夜身体还完好）
2. 主角会翘课，但不是灾难性地翘（他会在必要时翘课，但不能被退学）
3. 小说中不需要逐天追踪时间——跳到"一周后"或"三天过去了"是合法的叙事省略
4. 最关键的一点：**两条线错峰经营。** 降临期高强度布防时，法师活动仅限于上课和吃饭；冷却期准备时，法师可以更活跃地进行现实活动

## 三、分身身份的写作规范

### 3.1 核心 = 主角的延伸，不是第二人格

| ❌ 禁止写法 | ✅ 正确写法 |
|-----------|-----------|
| "分身对主角说……" | "主角通过分身感知到……" |
| "分身擅自做了决定" | "主角用分身的权限操作了地下城" |
| "分身有自己的想法" | "分身反馈的信息就是主角自己的感知" |
| "分身学会了……" | "主角掌握了分身的新能力……" |

**核心意象**：核心分身不是另一个人、不是另一个意识、不是分裂出来的副人格。

它更像是主角突然发现自己长出了**第三只手**——这只手在地下城深处，但控制它的是同一颗大脑、同一个意志。

### 3.2 感知描述的正确方式

| 正确描述 | 说明 |
|---------|------|
| "他感应到第一层的怪物在躁动" | 主角意识的一部分在地下城 |
| "他调整了陷阱的触发阈值" | 主角通过分身权限操作 |
| "他看到了入侵者的位置" | 核心感知 = 主角的视觉延伸 |
| "他感觉到源质的流速变快了" | 核心分身反馈的状态信息 |

**永远不要让主角和分身对话。** 分身不是交谈对象。

### 3.3 切换写法范例

**错误**：
> 李冠阳的意识沉入体内，核心分身的视角接管了感知。"有人进来了。"分身在他脑海中说道。

**正确**：
> 李冠阳闭上眼，把意识沉到深处。再睁开时——不是用眼睛看，而是用地下城的空气感知。第一层的入口处，有三人已经跨过了门槛。他能感觉到他们的脚步震动，就像脚步震颤传到自己身上一样。

## 四、"阴险防守"的叙事来源

### 4.1 不可使用"系统自动生成"

主角设计防守方案时，**不能出现"系统弹窗给解决方案"或"面板自动提示最优布防"**。

**不这么做：**
> 【系统提示：检测到入侵者，推荐在第二层入口设置减速陷阱+毒雾组合，预计效果：中】

**而是这样做：**
> 李冠阳靠在椅子上，脑子里翻出前世玩过的那款塔防游戏。
> 记得有一个经典搭配——减速区后面的交叉火力。那帮人当年被这套路坑得摔键盘骂娘。
> 他不打算照搬。地下城的地形和那款游戏的关卡不一样。但底层逻辑可以用……
> ……他又翻了翻论坛上网友对"最恶心陷阱"的讨论帖，里面几个回复让他眼前一亮。

### 4.2 前世记忆的运用规则

| 规则 | 说明 | 范例 |
|------|------|------|
| **碎片化** | 不是完整攻略，是散装记忆片段 | "我记得有个游戏在转角放宝箱怪，很阴" |
| **需要加工** | 不能直接套用到蓝星的地下城体系 | "那个游戏的宝箱怪设定是……我得改成适合我地下城的样子" |
| **需要验证** | 记忆中的套路不一定能在现实中生效 | "理论上应该能行，得实际测试一下才知道效果" |
| **有遗忘** | 不是无限记忆库 | "我记得好像还有个更阴的……但怎么想不起来了" |

### 4.3 网友建议的运用

主角在论坛/群聊里伪装成普通玩家提问：
- "你们觉得最恶心的陷阱是什么？"
- "那种会叫援军的怪物设定是不是很烦？"

**运用规则：**
- 不直接抄——"这个网友说的减速陷阱很阴，但我得改一下……在这个位置再加一个连锁触发"
- 融合多个建议——"A建议减速，B建议毒雾，C说两个一起更恶心——那就这么干"
- 不暴露目的——提问时伪装成"游戏讨论"，不让对方察觉自己是真的要布防

**核心**：方案来自**主角的思考**，不是系统喂给他的。读者看到的每一个布局设计，都是主角自己的谋划。

## 五、前20章节奏模板

> **重要前提：初期找不到其他地下城。**
> 
> 主角只在觉醒时进过一次地下城（觉醒坑），侥幸逃生后短期内不会回去（太危险，入口可能已被封锁）。
> 在信息混沌的环境下，其他地下城的位置没有公开信息，觉醒者圈子未形成，主角**没有渠道知道其他地下城在哪**。
> 
> 因此，前20章的**所有地下城活动都围绕主角自己的绑定城展开**——
> - 入侵者是主动找上门的（降临位置被发现/被觉醒者偶然撞见）
> - 主角的"战斗"以后台操控为核心，而非以法师身份出去探索
> - 法师身份在前20章主要负责现实活动、人际关系、为地下城经营提供情报支持
> - 法师线"去别人地下城打怪"的节点不早于第20章

### 5.1 章节节奏密度参考

| 章节区间 | 主调 | 降临期/冷却期 | 需要达到的目标 |
|---------|------|-------------|-------------|
| **1-3** | 觉醒认知 | 觉醒触发（无降临） | 让读者理解世界设定+主角双面板的机制 |
| **4-7** | 布局启动 | 第一次降临+第一次冷却 | 展示核心分身操控+首次入侵者来临+第一次实战防守 |
| **8-12** | 双线并行 | 第二次降临+冷却突发事件 | 冲突升级，外部势力开始关注 |
| **13-16** | 危机加码 | 第三次降临（更长） | 主角面临真正的经营危机和身份暴露风险 |
| **17-20** | 小高潮 | 降临期高潮阶段 | 一次成功的防守或突破，为下一阶段埋线 |

### 5.2 每章的信息释放节奏

| 章 | 释放信息 | 不释放信息 |
|----|---------|-----------|
| 1 | 世界背景（超凡出现）、主角现状 | 不直接展示全貌，只给碎片——地下城降临、觉醒发生 |
| 2 | 觉醒过程、前世记忆回溯 | 不带入"完整记忆"，只给核心记忆碎片 |
| 3 | 双面板认知、核心分身觉醒 | 先让主角知道他有"另一个视角"，不一定第一幕就完全理解 |
| 4 | 第一次降临（短时间尝试）、核心操控初体验 | 不展示全权限——只给最基础的操控项 |
| 5-6 | 入侵者误入主角地下城 | 入侵者视角：偶遇一处"异空间"入口，进去后发现这里和之前的异空间"不太一样"——但仅凭直觉疑惑，没有对比基准 |
| 7 | 冷却期经营、源质机制展示 | 不展示深层规则——只给收支框架 |
| 8+ | 逐渐展开世界+冲突 | 继续保持部分信息留白 |

- 每次降临的规模、时长、风险递增
- 信息逐步释放，不给"设定解释章"

## 六、常见写作陷阱检查表

### 6.1 设定层面

| 陷阱 | 检查标准 |
|------|---------|
| 成熟体系感太重 | 有没有出现不该在这个时期出现的机构/规则/定价/情报体系？ |
| 主角太懂 | 主角的所有知识是否都**明确标注来源**（前世记忆片段/论坛网友/自己推测）？ |
| 分身人格化 | 是否有让分身"说话"或"自己决定"的描写？ |
| 系统太智能 | 是否有"系统弹窗给思路"或"面板自动推荐方案"？ |
| 信息泄露太多 | 某句话是否让角色知道了不该在这个时期知道的事？ |

### 6.2 节奏层面

| 陷阱 | 检查标准 |
|------|---------|
| 连续高张力 | 是否连续两章都在高强度战斗/危机？是否需要插入一段冷却或日常过渡？ |
| 冷却期放空 | 冷却期是否"没什么事发生"？至少要有准备+意外或准备+埋钩子。 |
| 两条线失衡 | 是否连续多章只有一条线在推进？另一条线是否被完全遗忘？ |
| 切换过渡缺失 | 从降临期切换到冷却期时，是否有叙事过渡？还是突然切换？ |

### 6.3 战斗/经营层面

| 陷阱 | 检查标准 |
|------|---------|
| 阴险防守来自系统 | 主角设计的方案，来源是前世记忆/网友/自己推理，不是系统给的吗？ |
| 源质太容易 | 主角是否在源质获取上太过轻松？是否有明确标注消耗和收支平衡？ |
| 降临太频繁 | 每次降临间隔是否合理？冷却是否有成本？ |
| 法师面板被忽略 | 是否一直在写核心分身，法师面板的升级被遗忘了？ |
